# 墨问 CLI 工具 - 需求文档

## 1. 项目概述

### 1.1 项目背景

墨问是一个笔记平台，提供了 OpenAPI 供开发者使用。本项目旨在开发一个命令行工具，能够将本地 Markdown 文件批量上传到墨问平台，实现笔记的自动化迁移和备份。

### 1.2 项目目标

- 提供简单易用的 CLI 工具
- 支持批量上传 Markdown 文件
- 保留 Markdown 格式（富文本）
- 遵守墨问 API 的限制和规范

### 1.3 技术栈

- **开发语言**: Zig
- **基础框架**: zig-framework（复用 HTTP 客户端、文件系统等能力）
- **目标平台**: Windows, Linux, macOS

## 2. 功能需求

### 2.1 核心功能（MVP）

#### F1: 文件扫描
- **描述**: 扫描当前目录下的所有 `.md` 文件
- **范围**: 仅当前目录，不递归子目录
- **输出**: 文件路径列表

#### F2: Markdown 解析
- **描述**: 解析 Markdown 文件内容，构建语法树
- **支持的语法**:
  - 标题（H1-H6）: `# 标题`
  - 段落
  - 粗体: `**文本**`
  - 链接: `[文本](url)`
  - 引用: `> 文本`
  - 换行
- **暂不支持**:
  - 斜体: `*文本*` （与粗体标记冲突，MVP 暂不支持）
  - 代码块
  - 列表
  - 表格
  - 图片（需要额外的文件上传流程）
  - 嵌套样式（如粗体中的链接）

#### F2.1: 解析失败降级策略
- **策略**: 当 MD 解析失败时，降级为纯文本模式
- **行为**: 
  - 整个文件内容作为单个 paragraph
  - 不保留任何格式
  - 记录警告信息到结果报告
- **用户提示**: "警告: file.md 包含不支持的语法，已作为纯文本上传"
- **适用场景**:
  - 包含不支持的 Markdown 语法
  - 解析器遇到异常
  - 文件格式不规范

#### F3: 格式转换
- **描述**: 将 Markdown AST 转换为墨问 API 要求的 NoteAtom 格式
- **转换规则**:
  - Document → `type: "doc"`
  - Heading → `paragraph` + `bold` text
  - Paragraph → `type: "paragraph"`
  - Text → `type: "text"`
  - Bold → `marks: [{ type: "bold" }]`
  - Link → `marks: [{ type: "link", attrs: {href} }]`
  - Quote → `type: "quote"`

#### F4: API 上传
- **描述**: 调用墨问 API 上传笔记
- **端点**: `POST https://open.mowen.cn/api/open/api/v1/note/create`
- **认证**: Bearer Token（从配置文件读取）
- **限频**: 1 次/秒（自动控制）
- **配额**: 100 次/天

#### F5: 结果汇总
- **描述**: 收集每个文件的上传结果，生成汇总报告
- **内容**:
  - 总文件数
  - 成功数量
  - 失败数量
  - 失败文件列表及错误原因

#### F6: 配置管理
- **描述**: 从配置文件读取 API Key 等配置
- **配置文件路径**: 
  - Linux/macOS: `~/.mowen/config.json`
  - Windows: `%USERPROFILE%\.mowen\config.json`
- **配置项**:
  ```json
  {
    "api_key": "your-api-key-here",
    "api_endpoint": "https://open.mowen.cn",
    "timeout_ms": 30000
  }
  ```

### 2.2 命令行接口

#### 基本用法
```bash
mowen-cli [OPTIONS]
```

#### 选项
- `-h, --help` - 显示帮助信息
- `-v, --version` - 显示版本信息
- `-d, --dir <PATH>` - 指定目录（默认: 当前目录）
- `--config <PATH>` - 指定配置文件路径（默认: 见配置文件路径说明）
- `--auto-publish` - 自动发布笔记
- `--tags <TAG1,TAG2>` - 设置标签（逗号分隔）
- `--dry-run` - 预览模式，不实际上传
- `--verbose` - 详细输出

#### --dry-run 模式
- **行为**:
  - 执行所有步骤（扫描、解析、转换）
  - 不实际调用 API
  - 显示将要上传的内容预览
  - 显示转换后的 NoteAtom JSON 格式（使用 --verbose）
- **用途**:
  - 验证 MD 解析是否正确
  - 检查转换后的 NoteAtom 结构
  - 估算上传时间（文件数 × 1秒）
  - 调试和测试

#### 使用示例
```bash
# 上传当前目录的 MD 文件
mowen-cli

# 上传指定目录
mowen-cli -d ~/notes

# 自动发布并打标签
mowen-cli --auto-publish --tags "blog,tech"

# 预览模式
mowen-cli --dry-run
```

## 3. 非功能需求

### 3.1 性能要求

- **单文件处理时间**: < 2 秒（不含网络延迟）
- **批量上传**: 遵守 1 次/秒的限频要求
- **内存占用**: 合理使用内存，避免大文件导致内存溢出

### 3.2 可靠性要求

- **错误处理**: 网络错误、API 错误时给出清晰提示
- **容错性**: 单个文件失败不影响其他文件的上传
- **数据完整性**: 确保上传的内容与原文件一致

### 3.3 易用性要求

- **命令行参数**: 清晰易懂，符合 Unix 命令行习惯
- **错误信息**: 友好、具体，帮助用户快速定位问题
- **帮助文档**: 提供 `--help` 查看使用说明

### 3.4 安全性要求

- **API Key 管理**: 不硬编码，从配置文件读取
- **敏感信息**: 不输出到日志或控制台
- **配置文件权限**: 建议用户设置适当的文件权限

## 4. 约束条件

### 4.1 技术约束

- 使用 Zig 语言开发
- 复用 zig-framework 的能力
- 遵守墨问 API 规范

### 4.2 API 约束

- **限频**: 1 次/秒
- **配额**: 100 次/天
- **认证**: Bearer Token
- **端点**: `https://open.mowen.cn/api/open/api/v1/note/create`

### 4.3 数据约束

- **NoteAtom 结构**: 根节点必须是 `type="doc"`
- **递归结构**: NoteAtom 是递归定义的
- **节点类型**: 支持的节点类型有限（doc, paragraph, text, quote, bold, link, highlight 等）

## 5. 错误处理策略

### 5.1 错误类型

- **配置错误**: 配置文件不存在、格式错误、API Key 缺失
- **文件错误**: 文件不存在、读取失败、Markdown 格式错误
- **网络错误**: 连接失败、超时
- **API 错误**: 认证失败、限频、配额不足、服务器错误

### 5.2 处理策略

| 错误类型 | 处理方式 |
|---------|---------|
| 配置错误 | 立即退出，提示用户修复配置 |
| 文件错误 | 跳过该文件，记录到失败列表 |
| 空文件 | 跳过该文件，记录警告信息 |
| 网络错误 | 跳过该文件，记录到失败列表 |
| API 限频 (429) | 重试机制：等待 2s/5s/10s 后重试，最多 3 次 |
| API 配额不足 (403) | 停止上传，提示配额不足 |
| 其他 API 错误 | 跳过该文件，记录错误信息 |
| MD 解析失败 | 降级为纯文本模式，记录警告 |

### 5.3 重试策略

#### API 限频重试 (429)
- **第一次重试**: 等待 2 秒后重试
- **第二次重试**: 等待 5 秒后重试
- **第三次重试**: 等待 10 秒后重试
- **超过 3 次**: 跳过该文件，记录到失败列表

#### 网络错误重试
- **超时错误**: 重试 1 次（等待 3 秒）
- **连接错误**: 不重试，直接失败
- **其他网络错误**: 不重试，直接失败

### 5.4 错误信息

错误信息应包含：
- 错误类型
- 错误原因
- 建议的解决方案

示例：
```
错误: 配置文件不存在
路径: ~/.mowen/config.json
建议: 请创建配置文件并填入 API Key
```
